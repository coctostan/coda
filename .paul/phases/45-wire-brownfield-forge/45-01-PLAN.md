  <action>
    In the `forge` case of the command handler:

    1. Replace the raw `existsSync(codaRoot)` check with `detectBackdrop(projectRoot)`
    2. Add import: `import { scaffoldCoda, detectBackdrop } from '../forge';`
    3. Add import: `import { assembleScanContext } from '../forge';`
    4. Remove existing `import { scaffoldCoda } from '../forge';`

    Updated logic:
    ```typescript
    case 'forge': {
      const backdrop = detectBackdrop(projectRoot);

      if (backdrop.type === 'existing') {
        ctx.ui.notify('CODA is already initialized in this project. Use `/coda status` to see current state.');
        return;
      }

      // Scaffold .coda/ for both greenfield and brownfield
      scaffoldCoda(projectRoot);

      if (backdrop.type === 'brownfield') {
        // Assemble scan context for brownfield onboarding
        const promptsDir = join(dirname(dirname(__dirname)), '..', '..', 'modules', 'prompts');
        const scanCtx = assembleScanContext(projectRoot, promptsDir);
        const targetList = scanCtx.universalTargets.length > 0
          ? scanCtx.universalTargets.join(', ')
          : 'none detected';
        ctx.ui.notify(
          `✓ Brownfield project detected. \`.coda/\` directory created.\n\n` +
          `**Detected:** ${targetList}` +
          (scanCtx.sourceDir ? ` | Source: ${scanCtx.sourceDir}/` : '') +
          `\n\n**Next: Brownfield SCAN**\n` +
          `Read the detected files and run scan commands to gather evidence.\n` +
          `Use \`coda_create\` to write evidence files to \`.coda/forge/initial/onboarding/\`.`
        );
        return;
      }

      // Greenfield (unchanged)
      ctx.ui.notify(
        '✓ Project scaffolded. `.coda/` directory created with default configuration.\n\nNext: Create an issue...'
      );
      return;
    }
    ```

    Use `dirname` from the existing `path` import to resolve promptsDir relative to the commands.ts file location.

    Avoid:
    - Changing any other command cases
    - Modifying the greenfield message text (existing tests assert on it)
    - Adding new dependencies
  </action>
