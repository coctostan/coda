<objective>
## Goal
Wire `coda.json`'s `modules` section into the module system so projects can enable/disable modules and override blockThreshold per project. The registry already supports this via `RegistryConfig` — this phase adds the config loading plumbing from coda.json to the registry.

## Purpose
Without config integration, all modules are always enabled with default thresholds. Projects need the ability to disable modules (e.g., skip security for a docs-only project) and override thresholds (e.g., set tdd to 'none' for advisory-only mode).

## Output
- `CodaConfig` type extended with optional `modules` section
- Scaffold defaults updated with module config
- `module-integration.ts` gains `loadModuleConfig` helper to read coda.json
- Build-loop and phase-runner pass `codaRoot` to get project-specific config
- Tests validating config-driven module enable/disable and threshold override
</objective>

    - Export a new function:
      ```typescript
      export function loadModuleConfig(codaRoot: string): RegistryConfig {
        const configPath = join(codaRoot, 'coda.json');
        if (!existsSync(configPath)) return {};
        try {
          const raw = JSON.parse(readFileSync(configPath, 'utf-8')) as { modules?: Record<string, { enabled?: boolean; blockThreshold?: string }> };
          if (!raw.modules) return {};
          // Convert to RegistryConfig shape
          const modules: Record<string, { enabled: boolean; blockThreshold?: ... }> = {};
          for (const [name, cfg] of Object.entries(raw.modules)) {
            modules[name] = {
              enabled: cfg.enabled !== false,
              ...(cfg.blockThreshold ? { blockThreshold: cfg.blockThreshold as FindingSeverity | 'none' } : {}),
            };
          }
          return { modules };
        } catch { return {}; }
      }
      ```
    - Update `getModulePromptForHook` options type: add `codaRoot?: string`
    - When `codaRoot` is provided and `config` is not, call `loadModuleConfig(codaRoot)` to get config
    - Update `createModuleSystem` similarly: add optional `codaRoot` parameter as a third option. When provided and config is not, auto-load.

    Avoid: Making codaRoot required — it must remain optional for testing and backward compat. Avoid parsing the entire coda.json into CodaConfig — only extract the `modules` field. Avoid importing from L1/L2.
