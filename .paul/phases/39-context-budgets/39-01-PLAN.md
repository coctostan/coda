<objective>
## Goal
Add context budget management — types and assembly functions that let the context builder target a per-phase token budget, including/excluding whole sections by priority without mid-section truncation.

## Purpose
Without budgets, context grows unbounded. A mature project with large ref docs, many task summaries, and accumulated module findings blows past useful limits. Budget-aware assembly gives each phase the right amount of context.

- New `context-budgets.ts` with `ContextSection`, `ContextBudget` interfaces + `assembleWithinBudget()` + `estimateTokens()`
- New test file `context-budgets.test.ts`

## AC-1: Required sections always included
```gherkin
Given sections with a "required" priority that exceed the budget alone
When assembleWithinBudget is called
Then required sections are always included, even over budget
```

## AC-2: Priority-ordered inclusion
```gherkin
Given sections with required, high, medium, low priorities and a budget of 500 tokens
When assembleWithinBudget is called
Then required + high are included, medium/low omitted when budget tight
```

## AC-3: No mid-section truncation
```gherkin
Given a section that would push total past budget
When assembleWithinBudget evaluates it
Then the section is included whole or excluded whole — never truncated
```

## AC-4: Token estimation
```gherkin
Given a string of known length
When estimateTokens is called
Then it returns approximately chars/4 (ceiling)
```

## AC-5: Empty sections handled
```gherkin
Given an empty sections array
When assembleWithinBudget is called
Then it returns an empty string
```

## AC-6: Budget metadata returned
```gherkin
Given sections assembled within a budget
When assembleWithinBudget is called
```

  <name>Task 1: Create context-budgets.ts</name>
  <files>packages/coda/src/workflow/context-budgets.ts, packages/coda/src/workflow/index.ts</files>
  <action>
    Create new file `packages/coda/src/workflow/context-budgets.ts`:

    **Interface `ContextSection`:**
    - `label: string` — for logging (e.g., "ref-system (auth section)")
    - `content: string` — the actual content
    - `priority: 'required' | 'high' | 'medium' | 'low'`
    - `estimatedTokens: number` — rough estimate, caller can use `estimateTokens()`

    **Interface `BudgetResult`:**
    - `content: string` — assembled context
    - `usedTokens: number` — total tokens included
    - `includedCount: number` — sections included
    - `excludedCount: number` — sections omitted
    - `includedLabels: string[]` — labels of included sections
    - `excludedLabels: string[]` — labels of excluded sections

    **Const `PHASE_BUDGETS: Record<string, number>`:**
    - specify: 4000, plan: 6000, review: 4000, build: 4000, verify: 6000, unify: 8000

    **Function `estimateTokens(text: string): number`:**
    - Return `Math.ceil(text.length / 4)`
    - Simple chars/4 approximation

    **Function `assembleWithinBudget(sections: ContextSection[], targetTokens: number): BudgetResult`:**
    - Sort sections by priority: required (0) > high (1) > medium (2) > low (3)
    - Within same priority, preserve original order
    - Always include "required" sections regardless of budget
    - For high/medium/low: include whole section if `usedTokens + estimatedTokens <= targetTokens`
    - Never truncate a section — include or exclude whole
    - Return BudgetResult with assembled content and metadata

    Add JSDoc on all exports. Add module-level JSDoc.

    In `index.ts`, add exports:
    ```typescript
    export type { ContextSection, BudgetResult } from './context-budgets';
    export { PHASE_BUDGETS, estimateTokens, assembleWithinBudget } from './context-budgets';
    ```

    Avoid:
    - Modifying context-builder.ts (wiring happens when callers are updated)
    - Adding external dependencies
    - Actual tokenizer integration (chars/4 is intentionally approximate)
  </action>
  <verify>bun run tsc --noEmit</verify>
  <done>Types compile, functions exported. AC-1 through AC-6 structurally satisfied.</done>
</task>
