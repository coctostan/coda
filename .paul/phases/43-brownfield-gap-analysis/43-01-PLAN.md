- `GapDomain` interface and `GAP_DOMAINS` const with dependency ordering
- `assembleGapAnalysisContext()` function
- `writeGapAnalysis()` for persisting the gap analysis artifact
- Tests

<task type="auto">
  <name>Task 1: Add GAP ANALYSIS to brownfield.ts</name>
  <files>packages/coda/src/forge/brownfield.ts, packages/coda/src/forge/index.ts</files>
  <action>
    Add to the existing `brownfield.ts`:

    **Interface `GapDomain`:**
    - `name: string` — domain name (e.g., 'quality')
    - `description: string` — what to assess
    - `dependsOn: string[]` — domains that should be fixed first

    **Const `GAP_DOMAINS: readonly GapDomain[]`:**
    ```
    [
      { name: 'quality', description: 'CI pipeline, test framework, linting, type checking', dependsOn: [] },
      { name: 'security', description: 'Auth patterns, input validation, secrets, dependency vulnerabilities', dependsOn: ['quality'] },
      { name: 'architecture', description: 'Layer boundaries, module structure, god files, circular dependencies', dependsOn: [] },
      { name: 'data', description: 'Schema health, migration safety, query patterns', dependsOn: ['architecture'] },
      { name: 'documentation', description: 'README, API docs, inline docs, decision records', dependsOn: [] },
    ]
    ```

    **Interface `GapAnalysisContext`:**
    - `evidence: Array<{ module: string; body: string }>` — evidence from SCAN
    - `domains: readonly GapDomain[]` — domains to assess with dependency ordering
    - `evidenceCount: number`

    **Function `assembleGapAnalysisContext(codaRoot: string): GapAnalysisContext`:**
    - Read all evidence via readAllEvidence
    - Map to { module, body } pairs
    - Return with GAP_DOMAINS

    **Function `writeGapAnalysis(codaRoot: string, body: string): string`:**
    - Write to `.coda/forge/initial/GAP-ANALYSIS.md`
    - Add frontmatter: `{ title: 'Gap Analysis', generated_at: ISO timestamp }`
    - Create directory if needed
    - Return file path

    **Function `readGapAnalysis(codaRoot: string): { frontmatter: Record<string, unknown>; body: string } | null`:**
    - Read from `.coda/forge/initial/GAP-ANALYSIS.md`
    - Return null if not found

    Export new types and functions from forge/index.ts.

    Avoid:
    - Actually performing gap analysis (agent does that)
    - Modifying existing SCAN or SYNTHESIZE functions
  </action>
