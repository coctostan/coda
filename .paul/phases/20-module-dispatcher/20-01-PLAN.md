<objective>
## Goal
Implement the module dispatcher — assembles combined prompts for injection into phase sessions, parses structured findings from agent responses, and checks findings against block thresholds.

## Purpose
The dispatcher is the bridge between modules and the workflow. It provides prompt text for session injection (assemblePrompts) and structured results after the agent responds (parseAndCheckFindings). Per Decision D2, it NEVER touches agent sessions — the workflow layer handles that.

  <name>Task 1: Create dispatcher.ts with two-method API</name>
  <files>packages/core/src/modules/dispatcher.ts</files>
  <action>
    Create `packages/core/src/modules/dispatcher.ts` with:

    1. **Imports:** readFileSync from 'node:fs', types from './types', validateFindings from './finding-schema', SEVERITY_ORDER from './types', ModuleRegistry from './registry'

    2. **HookContext** interface:
       ```typescript
       interface HookContext {
         issueSlug: string;
         phase: string;           // string, not Phase type — avoids L2 import coupling
         submode: string | null;
         changedFiles?: string[];
         taskId?: number;
         planPath?: string;
       }
       ```

    3. **FINDINGS_OUTPUT_TEMPLATE** — the JSON requirement appended to all assembled prompts. Export as const string. Content from spec section "Agent Interaction Pattern" — the "## Required Output" block.

    4. **ModuleDispatcher** interface:
       ```typescript
       interface ModuleDispatcher {
         assemblePrompts(hookPoint: HookPoint, context: HookContext): string;
         parseAndCheckFindings(agentResponse: string, hookPoint: HookPoint): HookResult;
       }
       ```

    5. **createDispatcher(registry: ModuleRegistry, promptsDir: string): ModuleDispatcher**

       **assemblePrompts(hookPoint, context):**
       - Get modules from registry.getModulesForHook(hookPoint)
       - If no modules: return ''
       - For each module (priority order): read prompt file via readFileSync(registry.resolvePromptPath(mod, hookPoint), 'utf-8')
       - Build combined prompt:
         ```
         ## Module Analysis: {hookPoint}
         Issue: {context.issueSlug} | Phase: {context.phase}
         {if changedFiles: "Changed files: " + changedFiles.join(', ')}
         {if taskId: "Task: " + taskId}

         {module1 prompt content}

         {module2 prompt content}

         {FINDINGS_OUTPUT_TEMPLATE}
         ```
       - Return the assembled string

       **parseAndCheckFindings(agentResponse, hookPoint):**
       - Extract JSON from response via extractJsonFromResponse()
       - If no JSON found: return HookResult with empty findings, blocked=false (Decision #8)
       - Validate via validateFindings()
       - Inject module name into findings if empty (from the finding's self-reported module field)
       - For each valid finding, check if it exceeds its module's blockThreshold:
         - Look up module by finding.module via registry.getModule()
         - If found and exceedsThreshold(finding.severity, mod.blockThreshold): add to blockReasons
         - Format blockReason: "{MODULE} BLOCK: {finding.finding}" + assumption suffix if present
       - Return HookResult { hookPoint, findings, blocked, blockReasons, timestamp }

    6. **exceedsThreshold(severity: FindingSeverity, threshold: FindingSeverity | 'none'): boolean**
       - If threshold === 'none': return false
       - return SEVERITY_ORDER[severity] >= SEVERITY_ORDER[threshold as FindingSeverity]
       - Export this function (useful for testing and downstream consumers)

    7. **extractJsonFromResponse(response: string): unknown | null**
       - Look for ```json ... ``` fenced blocks first
       - If found, try JSON.parse on the content
       - If no fenced block, try to find a top-level JSON array (starts with '[')
       - If parsing fails or nothing found: return null
       - Do NOT export — internal helper

    JSDoc on all exports. No L1/L2 imports (HookContext uses string for phase/submode).
