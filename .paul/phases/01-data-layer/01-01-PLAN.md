<acceptance_criteria>

## AC-1: Record Read
```gherkin
Given a markdown file with YAML frontmatter and a body
When readRecord is called with the file path
Then it returns a typed frontmatter object and the body as a string
```

## AC-2: Record Write
```gherkin
Given a frontmatter object and a body string
When writeRecord is called with a file path
Then a valid markdown file is created with YAML frontmatter block and body content
```

## AC-3: Frontmatter Update
```gherkin
Given an existing record file
When updateFrontmatter is called with partial field updates
Then only the specified fields change, the body remains untouched
```

## AC-4: Section Parsing
```gherkin
Given a markdown body with multiple ## headings
When getSection is called with a heading name
Then it returns only the content under that heading (up to the next ## or EOF)
```

## AC-5: Section Mutation
```gherkin
Given a markdown body with sections
When appendSection is called with a new heading and content
Then the section is added at the end without touching existing sections
When replaceSection is called with an existing heading and new content
Then only that section's content changes, all others remain intact
```

## AC-6: Directory Listing
```gherkin
Given a directory containing .md record files
When listRecords is called with the directory path
Then it returns an array of file paths for all .md files in that directory
```

## AC-7: Type Safety
```gherkin
Given the TypeScript type definitions for IssueRecord, PlanRecord, TaskRecord, CompletionRecord, and ReferenceDoc
When used with readRecord<T> and writeRecord<T>
Then the compiler enforces correct field types at call sites
```
