<objective>

## Goal
Make the `unify→done` gate **evidence-based** instead of self-assertion-based, so UNIFY cannot complete unless at least one overlay or reference-doc update was actually produced on disk (or an explicit exemption was declared). Restructure the UNIFY runner prompt so the agent is told exactly which paths to list as evidence, with ceremony-aware relaxation for `refactor`/`chore`/`docs` issues.

## Purpose
F5 ("UNIFY produces zero compounding artifacts") is the headline fix for v0.10. The v0.9 live test showed the agent writing completion records with `system_spec_updated: true`, `reference_docs_reviewed: true`, `milestone_updated: true` — all assertions, zero corresponding artifacts. The overlay infrastructure (`readOverlay`, `writeOverlay`, `loadMergedPrompt`) is already wired and will auto-inject project context into future prompts the moment an overlay file exists with content. The only missing link is that nothing is forcing overlays to exist. Phase 54 adds that force at two points: the `unify→done` gate (enforcement) and the UNIFY runner prompt (clear instruction + declared schema for evidence reporting).

## Output
- `CompletionRecord` schema gains `artifacts_produced: { overlays: string[]; reference_docs: string[]; decisions: string[] }` and optional `exemptions?: { overlays?: string; reference_docs?: string; system_spec?: string }`
- `unify→done` gate validates that declared artifact paths exist on disk and contain non-empty content, with ceremony-driven relaxation for non-full-ceremony issue types
- UNIFY runner prompt restructured so ACTION 5 (completion record) carries the `artifacts_produced` schema inline and prepends a "Before writing the completion record, collect paths of every overlay / ref-doc you created or modified" block, with ceremony-aware exemption guidance
- New integration test `unify-artifacts-e2e.test.ts` proves the full gate pipeline blocks empty-evidence UNIFY and passes when artifacts exist on disk or exemptions are declared
- SUMMARY.md at phase UNIFY reconciling tasks → AC → decisions

</objective>
