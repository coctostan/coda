    1. Update `packages/coda/package.json`:
       - Add `"type": "module"` 
       - Update `pi.extensions` to point to `"./src/pi/index.ts"`
       - Ensure peerDependencies include `@sinclair/typebox`

    2. Rewrite `packages/coda/src/pi/types.ts`:
       - Remove our custom PiAPI interface entirely
       - Re-export `ExtensionAPI`, `ExtensionContext`, `ExtensionCommandContext` from `@mariozechner/pi-coding-agent`
       - Keep only CODA-specific types (CodaExtensionState, etc.)
       - JSDoc on all types

    3. Rewrite `packages/coda/src/pi/commands.ts`:
       - Import `ExtensionAPI` from `@mariozechner/pi-coding-agent`
       - `registerCommands(pi: ExtensionAPI, codaRoot: string): void`
       - Commands use `pi.registerCommand(name, { description, handler: async (args, ctx) => { ... } })`
       - Handler receives `args: string` (single string, not array) and `ctx: ExtensionCommandContext`
       - Use `ctx.ui.notify()` for output instead of console.log

    4. Rewrite `packages/coda/src/pi/tools.ts`:
       - Import `Type` from `@sinclair/typebox`
       - `registerTools(pi: ExtensionAPI, codaRoot: string): void`
       - Tools use `pi.registerTool({ name, label, description, promptSnippet, parameters: Type.Object({...}), execute(toolCallId, params, signal, onUpdate, ctx) { ... } })`
       - Each tool returns `{ content: [{ type: "text", text: ... }], details: {} }`

    5. Rewrite `packages/coda/src/pi/hooks.ts`:
       - `registerHooks(pi: ExtensionAPI, codaRoot: string): void`
       - `pi.on("before_agent_start", async (event, ctx) => { ... })` — returns `{ systemPrompt, message: { customType, content, display } }` or undefined
       - `pi.on("tool_call", async (event, ctx) => { ... })` — uses `event.toolName`, `event.input`, returns `{ block: true, reason }` or undefined
       - `pi.on("session_start", async (event, ctx) => { ... })` — initialize state, set status
       - State loaded from .coda/state.json via @coda/core loadState

    6. Rewrite `packages/coda/src/pi/index.ts`:
       - `export default function codaExtension(pi: ExtensionAPI) { ... }`
       - Calls registerCommands, registerTools, registerHooks
       - Re-exports CODA-specific types only

    7. Update `packages/coda/src/index.ts`:
       - Keep `export * from './pi'` but ensure no conflicting re-exports with pi-coding-agent types

    8. Update tests to use mock matching real ExtensionAPI shape:
       - Mock `pi.registerCommand`, `pi.registerTool`, `pi.on` matching real signatures
       - Test command registration count and names
       - Test tool registration count and names  
       - Test hook registration for before_agent_start, tool_call, session_start
       - Test tool_call handler blocks .coda/ writes

    Quick smoke test: `pi -e ./packages/coda/src/pi/index.ts` loads without errors
