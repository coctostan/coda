<acceptance_criteria>
## AC-1: Structural review failures produce deterministic revision instructions
```gherkin
Given a review-phase issue whose latest plan/tasks have missing AC coverage, invalid dependency ordering, or unreasonable file targets
When the review runner evaluates the issue in `submode: "review"`
Then it detects those structural failures without relying on LLM judgment, writes `.coda/issues/{slug}/revision-instructions.md`, and routes the state toward `submode: "revise"`
```

## AC-2: Revision cycles preserve loop state and history
```gherkin
Given a review-phase issue already carrying prior revision instructions and `submode: "revise"`
When the revise path completes and transitions back to `submode: "review"`
Then `loop_iteration` increments exactly once and the next review context includes prior revision history from disk
```

## AC-3: Clean review exits with approval
```gherkin
Given a review-phase issue whose structural checks pass and whose review result is approved
When the review runner completes the review path
Then it returns an approved outcome, updates the plan status to approved, and does not create new revision instructions
```

## AC-4: Exhaustion stops autonomous review safely
```gherkin
Given a review-phase issue whose `loop_iteration` has reached `max_review_iterations`
When the runner is asked to continue the review/revise loop
Then it returns an exhausted outcome and preserves the current review artifacts for human follow-up instead of re-entering the loop
```

## AC-5: Review and revise contexts are submode-aware
```gherkin
Given a review-phase issue with a plan, tasks, and optional revision-instructions history
When workflow context is requested for `review` versus `revise`
Then review context includes plan/tasks plus prior revision history, and revise context includes plan/tasks plus the active revision instructions artifact
```

## AC-6: Phase 10 changes keep workflow quality gates green
```gherkin
Given the Phase 10 implementation is complete
When the workflow test targets, full Bun test suite, and TypeScript typecheck are run
Then they pass without regressions
```
