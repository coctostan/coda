<task type="auto">
  <name>Task 2: Create finding-schema.ts with validation + tests</name>
  <files>packages/core/src/modules/finding-schema.ts, packages/core/src/modules/__tests__/finding-schema.test.ts</files>
  <action>
    **finding-schema.ts:**

    1. `validateFinding(raw: unknown): Finding | null`
       - Check `raw` is a non-null object
       - Required fields: `module` (string), `check` (string), `severity` (valid FindingSeverity), `finding` (string)
       - If `module` is missing, DON'T reject ‚Äî leave it for the caller to inject from context (lenient parsing per spec)
       - If `severity` is not a valid enum value, return null
       - If any other required field missing, return null
       - Optional fields: `file`, `assumption`, `recommendation`, `evidence` (strings), `hookPoint` (valid HookPoint)
       - Extra fields: ignored (not rejected)
       - Return typed Finding on success, null on failure

    2. `validateFindings(raw: unknown): Finding[]`
       - If raw is not an array, return []
       - Map each element through validateFinding
       - Filter out nulls (partial results OK ‚Äî don't lose valid findings)
       - Return Finding[]

    **finding-schema.test.ts:**

    8 test cases from spec:
    1. Finding with all fields validates correctly
    2. Finding missing optional fields validates (they're optional)
    3. Finding missing required field (check, severity, finding) returns null
    4. Invalid severity value returns null
    5. Array with mix of valid/invalid ‚Üí returns only valid ones
    6. Extra fields on finding ‚Üí ignored (not rejected)
    7. SEVERITY_ORDER comparison: critical > high > medium > low > info
    8. Empty array input ‚Üí empty array output

    Test style: Follow existing patterns in packages/core/src/state/__tests__/. Use bun:test, describe/test/expect.
  </action>
  <verify>Run `cd packages/core && bun test` ‚Äî all existing tests (80) plus new tests pass</verify>
