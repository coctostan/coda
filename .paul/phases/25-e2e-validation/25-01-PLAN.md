## AC-1: Full Prompt Assembly Path
```gherkin
Given a configured module system with security + tdd enabled
When assemblePrompts is called for each hook point
Then pre-plan returns security prompt content
And pre-build returns tdd prompt content
And post-task returns tdd prompt content
And post-build returns security + tdd prompt content (combined by priority)
And post-unify returns empty (no v0.3 modules at this hook)
```

## AC-2: Full Findings Path — Parse + Persist + Load + Summarize
```gherkin
Given an agent response containing JSON findings from a post-build hook
When parseAndCheckFindings processes the response
And persistFindings writes the result
And loadFindings reads it back
And summarizeFindings formats it
Then the full round-trip produces correct data at each step
And the summary matches the expected compact format
```

## AC-3: Security Block → Gate Check Integration
```gherkin
Given a critical security finding persisted to module-findings.json
When gatherGateData reads the findings file (as wired in Phase 22)
Then moduleBlockFindings > 0
And the build→verify gate returns { passed: false, reason: 'Module findings require attention' }
```

## AC-4: Config-Disabled Module → Silent Skip
```gherkin
Given config with tdd.enabled = false
When the full path runs for pre-build hook
Then no prompts are assembled for tdd
And no tdd findings are produced
And the gate has no tdd-related blocks
```

## AC-5: 'none' Threshold → Advisory Only
```gherkin
Given config with security.blockThreshold = 'none'
When a critical security finding is parsed
Then the finding is recorded (not discarded)
But the module does NOT block (blockReasons is empty)
And the gate passes (moduleBlockFindings = 0)
```

## AC-6: Verify/Unify Context Includes Findings
```gherkin
Given persisted module findings for an issue
When getPhaseContext is called for 'verify' phase
Then the context string contains '## Module Findings' with the summary
And when called for 'unify' phase, same inclusion
And when no findings exist, the section is absent
```

## AC-7: Clean Lifecycle — No Findings
```gherkin
Given an agent response with only INFO-level findings (no blocks)
When the full path runs
Then all findings are persisted
And moduleBlockFindings = 0
And the gate passes
And the summary shows counts but no (BLOCKED) marker
```

## AC-8: No Regressions
```gherkin
Given the existing 144 core + 197 coda tests
When the E2E test suite is added
Then all existing tests still pass and tsc --noEmit is clean
```
