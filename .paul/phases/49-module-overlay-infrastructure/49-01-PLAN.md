# Phase 49, Plan 01: Module Overlay Infrastructure

## Objective

Add per-project module overlay infrastructure so that projects can customize module behavior for plan/apply phases without modifying base prompts. Implements the two-layer prompt model from coda-spec-v7: default prompt (read-only) + project-local overlay (`.coda/modules/{module}.local.md`).

## Spec References

- `docs/coda-spec-v7.md` lines 1582–1604 (Module Project Overlays)
- `docs/v0.5/02-module-overlays.md` (detailed design)
- Decision D3 (v0.8): Overlay write via `coda_edit_body` (no new tool for v0.8)

## Deliverables

### D1: Core Overlay Module (`packages/core/src/modules/overlay.ts`) — NEW

**Types:**
- `OverlaySection` — union type: `'project_values' | 'validated_patterns' | 'known_false_positives' | 'recurring_issues'`
- `OverlayFrontmatter` — `{ module: string; last_updated: string; updated_by: string }`
- `OverlayData` — `{ frontmatter: OverlayFrontmatter; sections: Record<OverlaySection, string[]> }`

**Functions:**
- `readOverlay(codaRoot: string, moduleName: string): OverlayData | null`
  - Reads `.coda/modules/{moduleName}.local.md`
  - Parses YAML frontmatter + 4 known sections
  - Returns null if file doesn't exist
- `writeOverlay(codaRoot: string, moduleName: string, data: OverlayData): void`
  - Writes/overwrites `.coda/modules/{moduleName}.local.md`
  - Serializes frontmatter + sections
- `appendToOverlay(codaRoot: string, moduleName: string, section: OverlaySection, content: string, updatedBy: string): void`
  - Creates overlay file if it doesn't exist
  - Appends a bullet item to the named section
  - Updates `last_updated` and `updated_by` in frontmatter
- `loadMergedPrompt(defaultPromptPath: string, codaRoot: string, moduleName: string): string`
  - Reads default prompt from `defaultPromptPath`
  - Reads overlay from `.coda/modules/{moduleName}.local.md`
  - Returns `defaultPrompt + "\n\n## Project-Specific Context\n\n" + overlayBody`
  - If no overlay exists, returns default prompt unchanged
- `formatOverlayBody(data: OverlayData): string`
  - Renders overlay sections into markdown body

**Layer constraint:** L3 (modules). Uses `node:fs` + `node:path`. No L1/L2 imports.

## Out of Scope

- New modules (debt, state, errors, etc.) — separate effort
- Human feedback overlay tool (`coda_feedback`) — deferred to v0.9 (Decision D3)
- Prompt eval/tune cycle — post-v0.8
- Per-hook overlay files (single module-level overlay per spec)
- Gate automation — Phase 50

## Risk

- **Low:** Changes to `createDispatcher` signature are backward-compatible (optional param)
- **Low:** Overlay file format is simple YAML frontmatter + markdown — no new parser needed
- **Medium:** UNIFY/FORGE overlay instructions depend on agent following directions — but this is consistent with the existing instruction-based approach for all CODA actions
