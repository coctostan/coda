<objective>
## Goal
Create `ceremony.ts` with adaptive ceremony rules per issue type — controlling which phases are active, which modules fire, whether TDD is enforced, and whether human review is required. Config-driven with `coda.json` overrides.

## Purpose
v0.1-v0.3 treats every issue the same: full lifecycle ceremony. A bug fix doesn't need the same ceremony as a feature. A docs issue doesn't need TDD or autonomous verify. Adaptive ceremony gives each issue type the right depth.

## Output
- New `ceremony.ts` with `CeremonyRules` interface, `CEREMONY_DEFAULTS`, `getCeremonyRules()`
- New test file `ceremony.test.ts`
- Exported from `packages/coda/src/workflow/index.ts`
</objective>

<task type="auto">
  <name>Task 1: Create ceremony.ts with rules and lookup</name>
  <files>packages/coda/src/workflow/ceremony.ts, packages/coda/src/workflow/index.ts</files>
  <action>
    Create new file `packages/coda/src/workflow/ceremony.ts`:

    **Interface `CeremonyRules`:**
    - `reviewEnabled: boolean` — whether autonomous review phase runs
    - `reviewMaxIterations: number` — max review/revise cycles
    - `humanReviewDefault: boolean` — whether human review gate is on by default
    - `tddEnabled: boolean` — whether TDD enforcement applies
    - `verifyEnabled: boolean` — whether autonomous verify phase runs
    - `verifyMaxIterations: number` — max verify/correct cycles
    - `unifyFull: boolean` — full (spec delta + ref update) vs lightweight (record only)
    - `modulesEnabled: boolean` — whether module hooks fire
    - `specDeltaRequired: boolean` — whether spec delta is required

    **Const `CEREMONY_DEFAULTS: Record<string, CeremonyRules>`:**
    - `feature`: all enabled, reviewMaxIterations=3, verifyMaxIterations=3
    - `bugfix`: all enabled, humanReviewDefault=true, verifyMaxIterations=2
    - `refactor`: reviewEnabled=true but maxIterations=1, humanReviewDefault=false, unifyFull=false
    - `chore`: reviewEnabled=false, tddEnabled=false, modulesEnabled=false
    - `docs`: reviewEnabled=false, tddEnabled=false, verifyEnabled=false, modulesEnabled=false

    Values MUST match the spec table in docs/v0.4/03-ceremony.md exactly.

    **Function `getCeremonyRules(issueType: string, configOverrides?: Partial<CeremonyRules>): CeremonyRules`:**
    - Look up `issueType` in `CEREMONY_DEFAULTS`; fall back to `feature` if unknown
    - Merge with `configOverrides` via spread (overrides win)
    - Return merged rules

    Note: Use `Partial<CeremonyRules>` for the override param instead of a full CodaConfig type — keeps ceremony.ts decoupled from config types. The caller (phase-runner) will extract the relevant override slice from coda.json.

    Add JSDoc on all exports. Add module-level JSDoc.

    In `index.ts`, add exports:
    ```typescript
    export type { CeremonyRules } from './ceremony';
    export { CEREMONY_DEFAULTS, getCeremonyRules } from './ceremony';
    ```

    Avoid:
    - Importing config types from other modules (keep ceremony standalone)
    - Any integration into phase-runner or build-loop (out of scope)
    - Adding external dependencies
  </action>
  <verify>bun run tsc --noEmit</verify>
  <done>Types compile, function exported. AC-1 through AC-6 structurally satisfied.</done>
</task>
