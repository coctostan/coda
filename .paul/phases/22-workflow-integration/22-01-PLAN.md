## Goal
Wire the v0.3 module dispatcher into the existing workflow engine — module prompts fire at phase boundaries, the build→verify gate checks for module block findings, and the old v0.1 todd/walt module system is deleted in a single clean cut (Decision D4).

Connects the module infrastructure (Phases 18-21) to the runtime lifecycle. Without this wiring, the dispatcher and prompts exist but never fire. This makes modules an active part of the CODA issue workflow.

- Old coda modules (todd.ts, walt.ts) deleted, modules/index.ts rewritten
- Tests for new and modified functionality
</objective>


      ```
        if (!d.allPlannedTasksComplete) return { passed: false, reason: 'All planned tasks must be complete' };
      ```

    - Export `buildHookContext(issueSlug: string, phase: string, options?: { submode?: string | null; changedFiles?: string[]; taskId?: number; planPath?: string }): HookContext` — constructs HookContext from workflow parameters
    - Export `getModulePromptForHook(hookPoint: string, issueSlug: string, phase: string, options?: { config?: RegistryConfig; promptsDir?: string; submode?: string | null; changedFiles?: string[]; taskId?: number }): string` — convenience function that creates dispatcher, builds context, calls assemblePrompts, and returns the prompt string. This is the main entry point for build-loop and phase-runner.
    - Every exported function has a JSDoc comment
