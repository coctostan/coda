<objective>

## Goal
Make FORGE and issue-focus agent-callable by adding `coda_forge` and `coda_focus` tools to the Pi tool surface, and close the F7 write-gate regression observed in the v0.9 E2E findings by auditing Pi's mutation tool surface and adding a diagnostic integration test that mimics real Pi dispatch.

## Purpose
v0.9's live compounding test (Phase 52) proved CODA's compounding plumbing exists but the agent cannot enter the lifecycle autonomously: `/coda forge` and `/coda activate` are Pi slash commands, not agent-callable tools, so an agent in a bare workspace either stalls or bypasses the lifecycle by directly editing `state.json`. F7 showed the write gate — designed to block exactly that bypass — did not fire in Session 3 despite firing in Session 1. Phase 53 is the first piece of v0.10 "Close the Agent Loop": without agent entry points, nothing downstream (UNIFY artifacts, overlays, compounding) has a chance to trigger in autonomous use.

## Output
- Two new Pi-registered tools: `coda_forge`, `coda_focus` (12 tools total after landing)
- Shared `workflow/issue-activation.ts` helper; slash commands refactored to share logic with tools
- Write-gate integration test that simulates real Pi `tool_call` dispatch for `.coda/state.json` edits (F7 regression guard)
- Pi mutation-tool-surface audit baked into `pi/hooks.ts` (extend `isToolCallEventType` checks if new tool names exist)
- Diagnostic logging on gate firing (gated by `DEBUG=coda:*` env var)
- SUMMARY.md reconciling tasks → AC → decisions at phase UNIFY

</objective>
