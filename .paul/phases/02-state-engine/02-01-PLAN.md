<objective>
## Goal
Implement the M2 State Engine — the in-memory state machine that tracks position in the issue lifecycle, enforces phase transitions via gates, and persists state atomically to JSON.

## Purpose
The state engine is the coordination layer between data and tools. Without it, tools can't know what phase an issue is in, and invalid transitions can corrupt workflow. This is the second foundational layer of `@coda/core`.

## Output
- `packages/core/src/state/` directory with 4 source files and 3 test files
- `packages/core/src/index.ts` updated with L2 barrel export
- All functions passing unit tests via `bun test`
</objective>

<acceptance_criteria>

## AC-1: Valid Phase Transitions
```gherkin
Given a CodaState with phase set to a valid phase
When a transition to the next linear phase is attempted
Then the transition succeeds and the phase is updated
```

## AC-2: Invalid Phase Transitions Rejected
```gherkin
Given a CodaState with phase set to "specify"
When a transition to "build" (skipping phases) is attempted
Then the transition fails with a clear reason
```

## AC-3: Gate Enforcement
```gherkin
Given a gate for "specify→plan" that requires issueAcCount > 0
When the transition is attempted with issueAcCount = 0
Then the gate blocks with reason "Issue must have at least one acceptance criterion"
When attempted with issueAcCount = 2
Then the gate passes and transition succeeds
```

## AC-4: Atomic State Persistence
```gherkin
Given a CodaState object
When persistState is called
Then state is written via temp file + rename (atomic)
And loadState reads it back identically
```

## AC-5: Safe State Loading
```gherkin
Given a state file path where a .tmp file exists (failed prior write)
When loadState is called
Then the .tmp file is cleaned up
And the existing valid state file is loaded (or null if no state file)
```

## AC-6: Default State Creation
```gherkin
Given no existing state
When createDefaultState is called
Then it returns a CodaState with version 1, null focus_issue, null phase, tdd_gate locked, enabled true
```
