<objective>
## Goal
Add `getSectionsByTopics()` and `getSectionHeadings()` to the core data layer's section reader, enabling topic-based filtering of markdown sections for context-efficient retrieval.

## Purpose
Reference docs can be large (20+ sections). Loading entire docs into every phase session wastes context. Topic-based retrieval injects only the sections relevant to the current issue's topics — directly enabling the v0.7 context budget system.

## Output
- Two new exported pure functions in `sections.ts`
- Full test coverage in `sections.test.ts`
- Re-exported from `packages/core/src/data/index.ts`
</objective>

<acceptance_criteria>

## AC-1: Topic matching returns relevant sections
```gherkin
Given a markdown body with sections "## Authentication", "## Auth Middleware", "## Data Model"
When getSectionsByTopics is called with topics ["auth"]
Then it returns sections whose headings contain "auth" (case-insensitive substring)
And "## Data Model" is NOT included
```

## AC-2: Multiple topics use OR logic
```gherkin
Given a markdown body with sections for auth, api, and data
When getSectionsByTopics is called with topics ["auth", "api"]
Then sections matching EITHER topic are included
```

## AC-3: Overview always included
```gherkin
Given a markdown body with an "## Overview" section
When getSectionsByTopics is called with any topics
Then the "## Overview" section is always included in results
```

## AC-4: Empty topics returns all sections
```gherkin
Given a markdown body with multiple sections
When getSectionsByTopics is called with an empty topics array
Then all sections are returned (no filtering)
```

## AC-5: getSectionHeadings returns heading list
```gherkin
Given a markdown body with sections "## Overview", "## Auth", "## API"
When getSectionHeadings is called
Then it returns ["Overview", "Auth", "API"]
```

## AC-6: Edge cases handled
```gherkin
Given edge cases: no matching sections, body with no sections, case variations
When getSectionsByTopics or getSectionHeadings is called
Then each returns correct results without errors
```
