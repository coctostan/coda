<objective>
## Goal
Extend the module system with `init-scan` hooks on all 5 modules, update backdrop detection to support brownfield projects, and create evidence file management for brownfield FORGE.

## Purpose
Brownfield FORGE needs each module to scan its domain. The `init-scan` hook point already exists in HookPoint, but no modules register for it. This phase wires the hooks, detects brownfield projects, and provides evidence file I/O — the foundation for the SCAN phase.

## Output
- 5 modules gain `init-scan` hook entries in MODULE_DEFINITIONS
- `detectBackdrop` returns `brownfield` for projects with code but no `.coda/`
- New `evidence.ts` with `writeEvidence()` and `readEvidence()` for `.coda/forge/initial/onboarding/`
- Updated types and exports
</objective>

  <name>Task 2: Create evidence.ts for evidence file management</name>
  <files>packages/coda/src/forge/evidence.ts, packages/coda/src/forge/index.ts</files>
  <action>
    Create new file `packages/coda/src/forge/evidence.ts`:

    **Interface `EvidenceFrontmatter`:**
    - `module: string`
    - `scanned_at: string` (ISO timestamp)
    - `files_read: string[]`
    - `commands_run: string[]`

    **Interface `EvidenceRecord`:**
    - `frontmatter: EvidenceFrontmatter`
    - `body: string`

    **Const `EVIDENCE_DIR = 'forge/initial/onboarding'`**

    **Function `writeEvidence(codaRoot: string, moduleName: string, frontmatter: EvidenceFrontmatter, body: string): string`:**
    - Creates directory `.coda/forge/initial/onboarding/` if needed
    - Writes `EVIDENCE-{moduleName}.md` with YAML frontmatter + body
    - Uses `writeRecord` from @coda/core
    - Returns the file path

    **Function `readAllEvidence(codaRoot: string): EvidenceRecord[]`:**
    - Reads all `EVIDENCE-*.md` files from the onboarding directory
    - Parses each with `readRecord` from @coda/core
    - Returns array sorted by filename
    - Returns empty array if directory doesn't exist

    **Function `readEvidence(codaRoot: string, moduleName: string): EvidenceRecord | null`:**
    - Reads a single evidence file by module name
    - Returns null if not found

    Add JSDoc on all exports. Export from forge/index.ts.

    Avoid:
    - Importing from workflow layer (forge is L5, workflow is L6)
    - Adding external dependencies
  </action>
  <verify>bun run tsc --noEmit</verify>
  <done>Evidence functions compile, exported from forge index. AC-4, AC-5 structurally satisfied.</done>
