<objective>
## Goal
Implement the module registry for the CODA module system. The registry discovers enabled modules, resolves their definitions, and provides lookup by hook point — the "what modules exist and are active" layer.

## Purpose
The dispatcher (Phase 20) and workflow integration (Phase 22) need to know which modules fire at which hook points. The registry is the single source of truth for module definitions and their enabled/disabled state.

## Output
- `packages/core/src/modules/registry.ts` — createRegistry, MODULE_DEFINITIONS (security + tdd only), DEFAULT_THRESHOLDS, ModuleRegistry interface
- `packages/core/src/modules/__tests__/registry.test.ts` — 8 test cases from spec (adjusted for 2-module scope)
- `packages/core/src/modules/index.ts` — Updated barrel to include registry exports
</objective>

  <action>
    Create `packages/core/src/modules/registry.ts` with:

    1. **Import** types from './types': ModuleDefinition, ModuleHook, ModuleConfig, FindingSeverity, HookPoint

    2. **MODULE_DEFINITIONS** — Only 2 modules for v0.3 (Decision D3):
       ```
       security: name='security', domain='Security Patterns', version='1.0.0'
         hooks: pre-plan (priority 80, 'security/pre-plan.md'), post-build (priority 130, 'security/post-build.md')
       tdd: name='tdd', domain='Test-Driven Development', version='1.0.0'
         hooks: pre-build (priority 100, 'tdd/pre-build.md'), post-task (priority 100, 'tdd/post-task.md'), post-build (priority 200, 'tdd/post-build.md')
       ```
       Type: `Record<string, Omit<ModuleDefinition, 'enabled' | 'blockThreshold'>>`
       Export as const.

    3. **DEFAULT_THRESHOLDS**:
       ```
       security: 'critical'
       tdd: 'high'
       ```
       Type: `Record<string, FindingSeverity | 'none'>`
       Export as const.

    4. **RegistryConfig** interface (minimal — full CodaConfig comes in Phase 23):
       ```typescript
       interface RegistryConfig {
         modules?: Record<string, ModuleConfig>;
       }
       ```

    5. **ModuleRegistry** interface:
       - getEnabledModules(): ModuleDefinition[]
       - getModulesForHook(hookPoint: HookPoint): ModuleDefinition[]
       - getModule(name: string): ModuleDefinition | null
       - resolvePromptPath(module: ModuleDefinition, hookPoint: HookPoint): string

    6. **createRegistry(config: RegistryConfig, promptsDir: string): ModuleRegistry**
       - Iterate MODULE_DEFINITIONS
       - For each: check config.modules?.[name]?.enabled !== false (default = enabled)
       - Build ModuleDefinition with blockThreshold from config or DEFAULT_THRESHOLDS
       - getModulesForHook: filter by hookPoint match, sort by priority (ascending)
       - resolvePromptPath: use join(promptsDir, hook.promptFile) — import { join } from 'node:path'

    IMPORTANT: No imports from L1 (data) or L2 (state). Registry is L3-internal.
    JSDoc on every export. Follow style from types.ts.
  </action>
  <verify>Run `cd packages/core && bunx tsc --noEmit` — types must compile with zero errors</verify>
  <done>AC-1 through AC-7 type foundations satisfied. AC-8 pending barrel update (Task 3).</done>
