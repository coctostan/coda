       - parameters: action (required, enum get/set), key (optional string), value (optional any)

       - `set` updates a top-level key

       - `set` rejects unknown top-level key

       ```
       ## Task Completion Protocol
       After completing this task:
       1. Mark the task complete with coda_update (set status: "complete")
       2. Add a Summary section to the task with coda_edit_body
       3. Call coda_status to check for the next pending task
       4. If there are more pending tasks, read the next task with coda_read and begin it immediately
       5. If all tasks are complete, call coda_advance to move to the verify phase
       ```
    2. Only inject this instruction for non-correction tasks (correction tasks have a narrower scope).
    3. Add a test: call `buildTaskContext()` for a regular task → assert context includes "Task Completion Protocol" and the 5 steps. Call for a correction task → assert it does NOT include the protocol.

    1. Review `buildFindingsSubmissionInstruction()` (lines 165-172). It already exists and appends to module prompts. Improve the instruction to be more explicit about the JSON format:
       ```
       ## Findings Submission
       After completing this module analysis, you MUST call the `coda_report_findings` tool to submit your findings.

       Parameters:
       - hook_point: "{hookPoint}" (use this exact value)
       - findings_json: A JSON array of finding objects, each with:
         - check: string (what you checked)
         - severity: "critical" | "high" | "medium" | "low" | "info"
         - finding: string (what you found)
         - assumption: string (optional — what must be true for this finding to matter)

       If you found no issues, submit a single info finding: [{"check":"module_review","severity":"info","finding":"No issues found","assumption":"N/A"}]

       This step is REQUIRED — the build→verify gate depends on persisted findings.
       ```
    2. Add/update test: verify `buildFindingsSubmissionInstruction('post-build')` output includes "MUST call", "coda_report_findings", "hook_point", JSON format details, and the no-issues example.
