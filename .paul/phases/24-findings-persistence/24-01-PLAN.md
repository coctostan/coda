<objective>
## Goal
Implement findings persistence to `.coda/issues/{slug}/module-findings.json` and a `summarizeFindings()` function for compact cross-phase context. Wire summarized findings into the context-builder for verify and unify phases.

## Purpose
Without persistence, module findings are ephemeral — they only exist during the session that produced them. Persistence makes findings available across phases (e.g., verify sees post-build blocks) and enables the `moduleBlockFindings` gate wired in Phase 22 to actually read data. Summarization keeps cross-phase context within budget.

## Output
- `persistFindings()` + `loadFindings()` functions in module-integration.ts
- `summarizeFindings()` function for compact context strings
- context-builder.ts includes summarized findings in verify/unify context
- Tests for persistence, loading, summarization, and context wiring
</objective>

    ```typescript
    }
    ```

    **1b. persistFindings(codaRoot, issueSlug, hookResult):**
    - Path: `join(codaRoot, 'issues', issueSlug, 'module-findings.json')`
    - If file exists, read it, parse as ModuleFindingsData, append hookResult to hookResults array
    - If file doesn't exist, create with `{ issue: issueSlug, hookResults: [hookResult] }`
    - Create parent directory if needed (`mkdirSync recursive`)
    - Write atomically (stringify + writeFileSync)
    - JSDoc explaining the append behavior

    **1c. loadFindings(codaRoot, issueSlug):**
    - Read and parse module-findings.json
    - Return `ModuleFindingsData` or `{ issue: issueSlug, hookResults: [] }` if missing
    - Gracefully handle malformed JSON (return empty)

    **1d. summarizeFindings(hookResults):**
    - Group all findings across all hookResults by module name
    - For each module: count findings by severity, note if any hookResult was blocked
    - Format: `"security: 1 critical (BLOCKED), 2 info | tdd: 3 medium, 1 low"`
    - For blocked or high+ severity findings, include the finding text inline
    - For low/info, just show counts
    - Return empty string if no findings at all
    - Keep output concise — this goes into phase context which has a budget

    **1e. Export from workflow barrel:**
    - Add persistFindings, loadFindings, summarizeFindings to workflow/index.ts exports

    Avoid: Using coda_* tools for persistence — this is direct fs access in the workflow layer. Avoid over-formatting — summarizeFindings should be compact, not verbose.
