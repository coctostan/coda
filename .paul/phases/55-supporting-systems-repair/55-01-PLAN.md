## Goal
Close the three remaining v0.10 support-system gaps so the compounding engine has a clean perimeter: (F6) make `coda_run_tests` execute under any Pi runtime, not just Bun; (F7) harden the Pi-hook write gate against symlink, bash-redirect, and custom-tool bypass vectors enumerated in Phase 53's audit; (deferred #1) remove the legacy `human_review_default` field from `CodaConfig` and auto-migrate existing project configs to `gates`.

## Purpose
v0.10 Close the Agent Loop cannot ship Phase 57's re-validation of Script A until the supporting systems stop producing false negatives. Phase 54 landed the headline F5 fix (UNIFY evidence gate); Phase 55 removes the remaining three blockers tracked in `docs/v0.8/E2E-COMPOUNDING-FINDINGS.md` F6 and F7, plus the v0.8 D4 deferred cleanup. With these fixed, Phase 56 can strengthen lifecycle prompts against a perimeter we actually trust, and Phase 57's live E2E can run `coda_run_tests` end-to-end without a `ReferenceError: Bun is not defined` half-way through TDD.

## Output
- `codaRunTests` runs under both Bun and Node by spawning via `node:child_process.spawnSync` (platform-resolved `bun` executable), with Bun fast-path preserved when available and a clear error when `bun` is missing from PATH in a non-Bun runtime.
- `hooks.ts` blocks three additional bypass vectors: (a) symlinked paths whose `realpath` resolves into `.coda/`, (b) bash compound/subshell/here-doc writes targeting `.coda/`, and (c) custom (non-built-in) tool calls whose `input` payload references `.coda/` — with an explicit allow-list for `coda_*` tools.
- `CodaConfig` no longer declares `human_review_default`; `getDefaultConfig()` emits only `gates`. A **single shared** `loadCodaConfig(codaRoot)` exported from `tools/coda-config.ts` replaces the 4 duplicated file-private helpers in `tools/coda-advance.ts`, `tools/coda-status.ts`, `pi/commands.ts`, and `workflow/review-runner.ts` (the narrower helper in `pi/tools.ts` stays put — different shape, only reads test commands). The shared loader detects legacy configs carrying `human_review_default`, derives a `gates` block from `DEFAULT_GATE_MODES`, rewrites `.coda/coda.json` idempotently, and returns the migrated config. `resolveGateMode`'s backward-compat branch is removed.
- `.paul/phases/55-supporting-systems-repair/55-01-SUMMARY.md` at UNIFY reconciling the three tasks against AC and module dispatch results.
