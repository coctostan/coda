<objective>
## Goal
Add `getCarryForwardSummaries()` to the context builder — a dependency-aware carry-forward function that replaces blind recency with targeted context. Tasks with `depends_on` get summaries from their declared dependencies; correction tasks get source tasks + failure artifacts; tasks with no dependencies fall back to recency.

## Purpose
v0.1-v0.3 injects the last 2-3 task summaries into each BUILD task's context (recency-based). If task 5 depends on tasks 1 and 3 (not 4), it gets the wrong context. Dependency-based carry-forward gives each task exactly the context it needs.

## Output
- New `getCarryForwardSummaries()` function in `context-builder.ts`
- Exported from `packages/coda/src/workflow/index.ts`
- Full test coverage (5 spec tests + edge cases)
</objective>

    ```typescript
    export function getCarryForwardSummaries(
      codaRoot: string,
      issueSlug: string,
      currentTask: { frontmatter: TaskRecord; body: string },
      allTasks: Array<{ frontmatter: TaskRecord; body: string }>,
      maxSummaries: number = 3
    ): string
    ```

    **Logic (three paths, in priority order):**

    1. **Correction task path:** If `currentTask.frontmatter.kind === 'correction'` AND `currentTask.frontmatter.fix_for_ac` is set:
       - Find all completed tasks whose `covers_ac` includes `fix_for_ac`
       - Load the verification failure artifact via `loadVerificationFailure()`
       - Return source task summaries + formatted failure artifact
       - Use existing `getSourceTaskSummaries()` for the task summaries
       - Format the failure artifact inline (ac_id, failed_checks, source_tasks)

    2. **Dependency-based path:** If `currentTask.frontmatter.depends_on` has entries:
       - Filter `allTasks` to completed tasks whose ids are in `depends_on`
       - Use `getSourceTaskSummaries()` with those ids
       - Missing dependencies (id not found or not complete) are silently skipped

    3. **Recency fallback:** If no depends_on entries:
       - Get all completed task ids, take the last `maxSummaries`
       - Delegate to existing `getPreviousTaskSummaries()`

    Add JSDoc comment matching project conventions.

    In `index.ts`, add `getCarryForwardSummaries` to the context-builder re-export.

    Avoid:
    - Modifying existing `getPreviousTaskSummaries` or `getSourceTaskSummaries` signatures
    - Modifying `build-loop.ts` or `phase-runner.ts` (wiring is out of scope — that integrates when context budgets land in Phase 39)
    - Adding any external dependencies
    - Throwing exceptions (return empty string for all error paths)
