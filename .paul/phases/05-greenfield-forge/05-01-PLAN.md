<objective>
## Goal
Implement the v0.1 Greenfield FORGE flow — the `/coda forge` command that initializes a new project by scaffolding `.coda/`, generating reference docs, and creating a first milestone.

## Purpose
FORGE is the entry point for new CODA projects. It detects that no `.coda/` directory exists, runs a conversational interview, scaffolds the project structure, generates reference documents (ref-system.md, ref-prd.md), and creates the first milestone. In v0.1 this is a simplified greenfield-only flow — no brownfield, no DESIGN advisors, no CHALLENGE phase.

## Output
- `packages/coda/src/forge/scaffold.ts` — directory/file creation, coda.json defaults
- `packages/coda/src/forge/greenfield.ts` — orchestrator for the forge flow
- `packages/coda/src/forge/types.ts` — shared types for the forge layer
- `packages/coda/src/forge/index.ts` — barrel export
- Tests verifying scaffold structure, coda.json defaults, ref doc validity, and idempotency
</objective>
