## Goal
Create the brownfield SCAN orchestration — identifies universal targets in an existing project, reads them, assembles init-scan module prompts via the dispatcher, and returns the full scan context for the agent to act on.

## Purpose
The SCAN phase is the first step of brownfield FORGE. It tells the agent what to look at and how to produce evidence. The agent then reads files, runs commands, and writes evidence via `writeEvidence()`.

<task type="auto">
  <name>Task 1: Create brownfield.ts with scan orchestration</name>
  <files>packages/coda/src/forge/brownfield.ts, packages/coda/src/forge/index.ts</files>
  <action>
    Create new file `packages/coda/src/forge/brownfield.ts`:

    **Const `UNIVERSAL_SCAN_TARGETS: readonly string[]`:**
    ```typescript
    ['README.md', 'CONTRIBUTING.md', 'AGENTS.md', 'CLAUDE.md',
     'package.json', 'Cargo.toml', 'go.mod', 'pyproject.toml']
    ```

    **Const `UNIVERSAL_COMMANDS: readonly string[]`:**
    ```typescript
    ['git log --oneline -20', 'ls -la']
    ```
    (Note: `find src -type f | head -50` is dynamic based on detected source dir — handled in assembleScanContext, not as a static command)

    **Interface `ScanContext`:**
    - `universalTargets: string[]` — universal files that exist in the project
    - `sourceDir: string | null` — detected source directory (src, lib, app, cmd, pkg) or null
    - `universalCommands: string[]` — commands to run for universal evidence
    - `modulePrompts: string` — assembled init-scan prompts from all enabled modules

    **Function `getUniversalScanTargets(projectRoot: string): string[]`:**
    - Check each file in UNIVERSAL_SCAN_TARGETS against projectRoot
    - Return only files that exist

    **Function `getSourceDirectory(projectRoot: string): string | null`:**
    - Check ['src', 'lib', 'app', 'cmd', 'pkg'] in order
    - Return the first directory that exists, or null

    **Function `assembleScanContext(projectRoot: string, promptsDir: string, config?: RegistryConfig): ScanContext`:**
    - Call getUniversalScanTargets to find existing universal files
    - Call getSourceDirectory to find the source dir
    - Build universal commands list (including `find {sourceDir} -type f | head -50` if sourceDir exists)
    - Create a module registry via createRegistry(config ?? {}, promptsDir)
    - Assemble init-scan prompts via createDispatcher(registry).assemblePrompts('init-scan', context)
    - Return ScanContext

    Add JSDoc on all exports. Add module-level JSDoc.
    Export from forge/index.ts.

    Avoid:
    - Actually executing commands (that's the agent's job)
    - Reading file contents (that's the agent's job — we just identify which files exist)
    - Any dependency on L6 workflow layer
  </action>
  <verify>bun run tsc --noEmit</verify>
  <done>Functions compile, exported. AC-1 through AC-4 structurally satisfied.</done>
</task>
