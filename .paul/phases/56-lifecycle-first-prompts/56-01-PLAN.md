<objective>
## Goal
Make CODA's injected guidance strong enough that an agent naturally enters and stays in the lifecycle: bootstrap with `coda_forge` when needed, create and focus an issue before building code, and use prompt-provided tool guidance instead of reading CODA source to figure out what to do next.

## Purpose
Phase 53 gave the agent the missing entry tools (`coda_forge`, `coda_focus`), and Phase 55 cleaned up the perimeter/runtime/config foundation. Phase 56 now has one job: make the agent actually use that tool surface without nudging. If the prompts and next-action hints stay weak, Phase 57's re-validation will still show the same F3/F4 behavior from `docs/v0.8/E2E-COMPOUNDING-FINDINGS.md`: build-first behavior and source spelunking to learn tools.

## Output
- `before_agent_start` injects a compact CODA bootstrap prompt when no issue is focused, with explicit issue-first instructions, a minimal tool manifest, a one-paragraph lifecycle overview, and an inline `coda_create(...)` → `coda_focus(...)` example.
- Focused lifecycle prompts in `phase-runner.ts` and `build-loop.ts` explicitly reinforce the current phase, the next expected CODA action, and the rule to trust injected guidance/tool descriptions before reading CODA source.
- `coda_status`, `coda_forge`, and `focusIssue()` return concrete, tool-oriented `next_action` strings aligned with the issue-first lifecycle.
- Tests pin the new guidance across unfocused bootstrap, focused phase prompts, BUILD task protocol, and status/entry-tool handoffs.
- `.paul/phases/56-lifecycle-first-prompts/56-01-SUMMARY.md` at UNIFY reconciling prompt changes, wording decisions, and verification results.
</objective>
