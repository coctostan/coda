## Design

### Execution model

The outer agent (this session) drives CMUX, playing the PM role:
- `cmux send coda "..."` — types developer-like messages
- `cmux read-screen coda` — observes inner agent response
- Direct filesystem reads — verifies artifacts independently

The inner agent (Pi + CODA extension) is the system under test:
- Receives CODA-injected prompts at phase boundaries
- Uses `coda_*` tools to navigate the lifecycle
- Builds actual code

### Script A flow

1. **Setup** — Create bare project, start Pi+CODA in CMUX pane
2. **FORGE** — Natural onboarding ("I want to set up this project with CODA...")
3. **Issue 1: URL Shortener**
   - SPECIFY → PLAN → BUILD (TDD) → VERIFY → UNIFY → DONE
   - Key: Does UNIFY produce meaningful overlays?
4. **Issue 2: Add Analytics**
   - Full lifecycle again
   - Key: Does knowledge from Issue 1 carry forward?
5. **Functional verification** — curl tests against running server
6. **Teardown + Findings report**
